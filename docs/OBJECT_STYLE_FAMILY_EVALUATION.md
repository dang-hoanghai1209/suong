# Object style-family evaluation

This benchmark compares a small controlled role set across Iconify's `mdi`, `material-symbols`, and `boxicons` collections. It is an evaluation workflow, not a production-ingestion policy. Binary outputs stay under the external library's `style_family_evaluation/` directory and every evaluation record remains production-ineligible.

The benchmark uses the fixed `#332821` background, identical normalization, warm foreground colors, crop/padding rules, output sizes, object slots, shadows, and unchanged Tella character assets. Each family receives two renders:

- `normalized`: source geometry with only safe color, alpha, crop, and size normalization.
- `tella_adapted`: the same geometry plus warm palette harmonization and round line caps/joins where the SVG genuinely uses strokes.

The adapted mode deliberately does not redraw paths. Most tested icon families encode apparent outlines as filled geometry, so this lightweight adaptation cannot turn them into hand-drawn story illustrations.

Run artifacts are generated by `tella.object_library.style_evaluation` using the shared `ObjectIngestionService` cache. The production object manifest is never written by the evaluator.

## Provisional decision

The current human-review decision is `NEEDS_STYLE_ADAPTATION`; expansion to 50–80 objects is not approved. MDI is the strongest general control, Boxicons is the most useful restricted fallback for ticket/popcorn outing props, and Material Symbols is not recommended because of coverage gaps and strong system-icon character. See `configs/object_library/style_family_policy.json` for the evaluation-only policy and thresholds.

The next smallest experiment is a controlled comparison of 12–15 story-rich physical props from a more illustrative source (including Noun Project when credentials are available), plus one stronger but bounded Tella adaptation prototype. It should repeat the same character-context benchmark before any broad ingestion.
